# KBO 기록 크롤러 및 Vue3 시각화 계획

## 목표

- KBO 기록실 타자 기본 기록 페이지의 전체 데이터를 크롤링한다.
- 크롤링한 데이터를 MySQL `kbo` 데이터베이스에 적재한다.
- 적재된 데이터를 Vue3 웹 화면에서 테이블 형태로 조회하고 시각화한다.
- Python, SQLModel, Vue3를 중심으로 최소 기능부터 동작하게 만든다.

## 현재 전제

- 대상 페이지: `https://www.koreabaseball.com/Record/Player/HitterBasic/Basic1.aspx?sort=HRA_RT`
- DB: MySQL
- DB 위치: WSL 로컬 `localhost:3306`
- DB 이름: `kbo`
- DB 사용자: `jys`
- DB 비밀번호는 `.env`에만 저장한다.
- 백엔드와 프론트엔드는 같은 저장소 안에서 관리한다.

## 성공 기준

- 크롤러를 실행하면 KBO 타자 기본 기록 데이터를 누락 없이 수집한다.
- 수집 데이터가 SQLModel 모델을 통해 MySQL에 저장된다.
- 동일 선수를 중복 적재하지 않거나, 실행 시점 기준으로 갱신 가능하다.
- API를 통해 적재된 기록 목록을 조회할 수 있다.
- Vue3 화면에서 기록 데이터를 테이블로 볼 수 있다.
- `.env`만 수정하면 DB 접속 정보를 변경할 수 있다.

## 기술 방향

- 크롤링: KBO 페이지 구조를 확인한 뒤 가장 단순하고 안정적인 방식 선택
  - 정적 HTML로 충분하면 `requests` + `BeautifulSoup`
  - JavaScript, postback, pagination 처리가 필요하면 `playwright`
- DB 모델: `SQLModel`
- DB 드라이버: MySQL 호환 드라이버 사용
- 설정: `.env`
- API 서버: Python 기반 경량 API
- 프론트엔드: Vue3

## 환경 변수 계획

`.env`에는 아래 값을 둔다.

```env
DB_HOST=localhost
DB_PORT=3306
DB_NAME=kbo
DB_USER=jys
DB_PASSWORD=<local-password>
```

필요하면 구현 단계에서 `DATABASE_URL`을 조립해서 사용한다.
